Key features
Live detection — Haar Cascade processes every webcam frame in real time
Auto-retraining — adding a new user immediately retrains the KNN model on all registered faces
No duplicate logging — roll number check prevents a person being marked more than once per session
Daily CSVs — one Attendance-MM_DD_YY.csv per day, ready for spreadsheet export
No cloud required — fully local, runs on any machine with Python and a webcam
Flask routes
GET / # dashboard — today's table GET /start # open webcam, take attendance POST /add # register new user # → capture 50 face samples # → retrain KNN → save .pkl

Model training
# Flatten each 50×50 face image into a 7,500-dim vector for user in os.listdir('static/faces'): for img_path in os.listdir(f'static/faces/{user}'): img = cv2.imread(img_path) faces.append(cv2.resize(img, (50, 50)).ravel()) labels.append(user) knn = KNeighborsClassifier(n_neighbors=5) knn.fit(faces, labels) joblib.dump(knn, 'static/face_recognition_model.pkl')

Limitations & roadmap
Single face per frame

Only the first detected face is identified — multi-person support is planned.

Flat feature vectors

50×50 flattening discards spatial structure; FaceNet / DeepFace embeddings would improve accuracy.

No anti-spoofing

A printed photo fools the model — liveness detection is on the roadmap.

Lighting sensitivity

Haar Cascade degrades in low light; an SSD or MTCNN detector would be more robust.

Local-only storage

Firebase or PostgreSQL integration planned for multi-device deployments.

Email reports

Auto-send daily attendance CSV to admin after session close — coming soon.
